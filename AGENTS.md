# AGENTS.md

Read **`specifications.md`** first — it defines the design system this site is built to. Match it; don't add to it.

## Stack
- Next.js 16.2.9 (App Router, Turbopack) · React 19.2 · TypeScript 5.5+
- Tailwind 3.4 with custom tokens · ESLint 9 (flat config)
- pnpm 9.0.0 only — `pnpm-lock.yaml` is committed; do not use npm or yarn
- Node `>=20.9.0` (see `engines` in `package.json`)

## Commands
| Task | Command |
|---|---|
| Dev server (Turbopack) | `pnpm dev` |
| Production build + typecheck | `pnpm build` |
| Lint | `pnpm lint` |
| Start production server | `pnpm start` |

**No test command exists.** `pnpm test` will fail. `pnpm build` runs `tsc` and is the typecheck.

Verification order after changes: `pnpm lint && pnpm build`.

## Client vs server components
- **Client** (`"use client"`): `app/projects/page.tsx`, `app/components/{Navbar,HeroDashboard,BarChart,StatCounter,Reveal,ContactForm}.tsx`
- **Server**: everything else, including `app/page.tsx` and `app/services/page.tsx`
- Add `"use client"` to any new component using hooks or event handlers

## Design system (from `specifications.md`)
- **Tokens** (use these Tailwind classes, never raw hex): `bg-paper`, `bg-paper-raised`, `text-ink`, `text-ink-muted`, `border-ink`, `border-rule`, `bg-signal`, `text-signal`
- **Borders**: 2px ink rules separate sections (`border-2 border-ink` + `<div className="hard-rule" />` between blocks)
- **Shadows**: `shadow-hard` (3 3 0), `shadow-hard-sm` (2 2 0), `shadow-hard-lg`, `shadow-hard-signal` — never soft drop shadows
- **Typography**:
  - Headlines → `font-display` (Playfair Display)
  - Body → `font-sans` (IBM Plex Sans) — set on `<body>` in `layout.tsx`
  - Labels, metadata, nav accents, code → `font-mono` (IBM Plex Mono) at `text-[10px] uppercase tracking-widest2`
- **Signal green** (`#00c853`) is reserved for: CTAs, active states, result metrics, key numbers. Do not sprinkle it on decorative elements.
- **§ numbering is continuous 01–13** across Home / Services / Projects. Do not restart per page.
- **Spacing**: 8px rhythm (`py-20`/`py-24` for major sections, `gap-4`/`gap-6` for cards)

## Gotchas an agent would miss
- **React 19 removed the global `JSX` namespace.** If you reference `keyof JSX.IntrinsicElements` or similar, add `import type { JSX } from "react"` at the top. See `app/components/Reveal.tsx:4`.
- **Next.js 16 removed `next lint`.** The `lint` script runs `eslint .` directly. The flat config in `eslint.config.mjs` spreads `eslint-config-next/core-web-vitals` directly (do not wrap with `FlatCompat`).
- **Two ESLint rules are intentionally disabled** in `eslint.config.mjs`: `react/no-unescaped-entities` (we use `'` and `&apos;` freely) and `@next/next/no-html-link-for-pages` (we use `href="#contact"` anchors).
- **Import paths differ per file**:
  - `app/page.tsx` → `./components/Foo`
  - `app/services/page.tsx`, `app/projects/page.tsx` → `../components/Foo`
  - Path alias `@/*` is configured in `tsconfig.json` but not used — match the existing relative imports.
- **Hero dashboard is hidden on mobile** (`hidden md:block` on the wrapper at `app/page.tsx:~142`).
- **`.opencode/` is gitignored** — it's the opencode CLI's local state, not a project artifact. Don't commit it; don't worry if it's missing.
- **`next-env.d.ts` is auto-generated** by Next.js but committed by convention. Don't edit it manually.
- **`tsconfig.tsbuildinfo`** is generated by `tsc` and gitignored.
- **`specifications.md`** is the design source of truth. If a styling decision feels arbitrary, check there before improvising.

## Editorial voice (for copy changes)
- Direct, second-person, written. Reads like a technical journal, not a marketing brochure.
- No exclamation points, no superlatives, no "we're the best" language.
- Numbers: state the receipt (e.g. "CAC down 41% in two quarters"), not the boast.
